## 设计哲学

> ### [EN](#philosophy)

1. 极高的 token 消耗是设计缺陷，而不是荣誉勋章。
2. 付给 Anthropic 或者 OpenAI 的账单不反映你的生产力。
3. 使用更多 token 不仅意味着成本增加，也意味着时间消耗增加。
4. 使用最为直观的概念和术语。
5. 花哨的富文本系统提示词是一种自我安慰。只使用必要的语义化格式。
6. 上下文是一种稀缺资源，必须被严格控制使用。
7. 我们必须假设增加上下文总是使模型变蠢，除非其价值超越了成本。
8. 感知力在 Agentic Coding 中格外重要。用户应该如手写代码一般了解他们的代码库。

## Philosophy

1. High token consumption is a clear defect in system structure. Not a badge of honor.
2. The bill to Anthropic or OpenAI is not a reflection of your productivity.
3. More token is not solely an increase in cost, and, moreover, a waste of time.
4. Intuitive and transparent terminologies.
5. System prompts with rich format is merely a self-appeasement. Use only necccesary semantic formatting.
6. Context is a scarce resource. Its usage must be managed.
7. We must always assume more context always compromises the model. Unless its value outweighs the cost.
8. Awareness is even more important in agentic coding. The user should know their codebase as if they wrote it by hand.
